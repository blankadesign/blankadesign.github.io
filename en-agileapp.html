AgileSprint App

Mobile | Responsive | Prototype

We designed a project managment tool which connects to Trello's API, allowing scrum teams to keep track of every task status of a sprint.


The Project

As teams grow, keeping track of each sprint's progress gets complex. Walls full of Post-It's do not escalate so well.

Trello helps to organize work but updating it takes time and it does not show progress in a visual manner.

Through our research, we came to realize that Trello's API could be used to create an extension.

Objectives

  • Simple and easy to to use.
  • Completely responsive
  • Fun usage to drive it's adoption

One Design Sprint

For this project we used Google Ventures' one-week, sprint methodology. We dedicated to research, to planning, wireframing and visual design one day each. We built a working MVP and reserved the last to testing and validation.




How it Works


The app has 2 main views: Check-in and Dashboard.

At Check in you can update every assigned task status. If you choose "blocked" you can write a message for the rest of the team.

The message is published as a comment in its Trello task through the API.

Desktop Version


Dashboard's cards use color classification iconography allowing a quick identification and marking of every status.

To take andvantage of bigger screen sizes, both sections are shown in the same view.



Mobile Version

We used Flinto to build an interactive prototype of the mobile app. Flinto allows a glimpse of how the native app interaction would feel. Everybody loved it!
