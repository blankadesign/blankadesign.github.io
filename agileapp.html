AgileSprint App

Mobile | Responsive | Prototype

Diseñamos una aplicación para project managers que se conecta a la API de Trello permitiéndole a los miembros de los equipos scrum actualizar el estado de cada tarea de un Sprint.


El Proyecto

A medida que los equipos crecen, se vuelve más complejo llevar un registro del progreso de cada Sprint. Los tableros llenos de Post-Its no escalan bien.

Trello ayuda a organizar el trabajo pero no muestra el progreso de manera visual y actualizarlo quita tiempo.

Durante nuestra investigación, nos dimos cuenta que la API de Trello nos sería supremamente útil para construir una extensión.

Objetivos

  • Facilidad de uso y simplicidad
  • Completamente responsive
  • Divertida de usar para impulsar su adopción

Un Sprint de Diseño

Para este proyecto seguimos la metodología de Sprint de Google Ventures. Dedicamos un día completo a la investigación, otro a planeación, bocetos y wireframes; otro al diseño de la interfaz; uno más a la construcción del prototipo (MVP) y el último a la validación.




Cómo Funciona


La aplicación tiene 2 vistas principales: Check in y Dashboard.

En Check in, puedes actualizar el estado de cada tarea que te has asignado. Si eliges “bloqueado” puedes redactar un mensaje para el resto del equipo.

El mensaje se publica como un comentario en su tarea de Trello a través de la API.

Versión Escritorio


Las cartas del Dashboard usan iconografía de clasificación por color que permite una rápida indentificación y marcación de cada estado.

Para aprovechar el espacio de las pantallas más grandes, ambas secciones son visibles en la misma vista.



Versión para Móviles

Utilizamos Flinto para construir un prototipo interactivo de la aplicación en móviles. Flinto nos permite dar la sensación de esta interactuando con una aplicación nativa. ¡A todos les encantó!.
